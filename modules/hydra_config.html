Functions

duplicate_line(direction)
Duplicate the current line and place it above or below.

Parameters:

  • direction The direction to place the duplicated line ("up" or "down").
log_level_str(level)
Converts a numeric highlight level value to a string compatible with `vim.api.nvim_echo`.

Parameters:

  • level The numeric value of the highlight level.

Returns:

    The string corresponding to the highlight level, or "info" if the value is invalid.
echo(message, highlight, redraw)
Displays a message in Vim.

Parameters:

  • message The text of the message to be displayed.
  • highlight (optional) The highlight group to be used for the message. Default is "Normal".
  • redraw (optional) Forces a redraw before displaying the message. Default is false.
notify(message, level, options)
Displays a notification in Vim.

Parameters:

  • message The text of the notification to be displayed.
  • level (optional) The level of the notification. Default is `vim.log.levels.INFO`.
  • options (optional) Additional options for the notification.
install_user_config(remote, branch)
Generates the user config. Checks if the user configuration already exists. If it doesn't exist, clones the HydraVim configuration template repository and installs it.

Parameters:

  • remote (optional) Remote of the repository to clone from.
  • branch (optional) Branch to clone from.
get_lsp_name()
Gets the active LSP (Language Server Protocol) name for the current filetype.

Returns:

    The name of the LSP or an empty string if no LSP is active for the filetype.
read_file(path, mode)
Reads the contents of a file.

Parameters:

  • path The path to the file to be read.
  • mode The file opening mode (e.g., "r" for reading).

Returns:

    The contents of the file as a string, or nil on error.
write_file(file, contents, mode)
Writes contents to a file.

Parameters:

  • file The path to the file to be written.
  • contents The string to be written to the file.
  • mode The file opening mode (e.g., "w" for writing).
generate_lockfile()
Generates the lockfile for plugins. This function generates a lockfile containing information about the plugins and their versions/commits.
